# 春辉综合服务部 微信小程序 — 执行计划

## 项目概述
基于 design.md 设计文档，开发一个面向乡镇/县城下沉市场的本地生活服务小程序，整合保洁清洗、养老陪护、搬家安装、家政月嫂、婚姻介绍、招聘求职等高频生活服务。

## 核心特征
- **四方角色**: 普通用户、工人/师傅、合作商、管理员
- **技术栈**: 微信小程序原生框架 + 微信云开发(CloudBase) + 微信支付 + 腾讯位置服务
- **页面数量**: 首页、全部服务、用户中心、工人中心、合作商中心、管理员后台、工人详情、订单详情、即时通讯、发布信息 等 15+ 页面
- **核心工作流**: 浏览→预约→接单→支付→评价→沟通 完整闭环

## 执行阶段

### Stage 1: 深度分析与核心文档编写
- **目标**: 全面分析设计文档，产出核心工作文档、项目记忆、技术方案
- **子任务**:
  1. 需求分析与功能拆解 — 梳理所有页面、组件、接口
  2. 数据库Schema设计 — 设计云数据库集合结构
  3. 项目结构与文件规划 — 定义小程序目录结构和页面清单
  4. 核心工作文档编写 — 编写 core_work.md（技术方案、API清单、组件清单）
  5. 项目记忆文档编写 — 编写 project_memory.md（设计决策、上下文、约束）
  6. 开发说明文档编写 — 编写 dev_guide.md（开发流程、规范、部署指南）
- **技能**: vibecoding-general-swarm（Stage 2 开始加载）

### Stage 2: 项目骨架搭建与云开发配置
- **目标**: 搭建小程序项目骨架，配置云开发环境
- **子任务**:
  1. 初始化小程序项目结构（pages、components、cloud/functions、images）
  2. 配置 app.json（页面路由、底部Tab、导航栏、下拉刷新等）
  3. 配置 app.js/app.wxss（全局样式、颜色变量、云开发初始化）
  4. 创建云数据库集合（users、workers、orders、messages、posts、banners、categories 等）
  5. 编写公共云函数（login、getOpenId、uploadImage 等）
  6. 编写工具函数库（utils: 日期格式化、距离计算、权限检查、数据请求封装）
- **技能**: vibecoding-general-swarm

### Stage 3: 首页开发（综合生活馆）
- **目标**: 实现首页所有模块
- **子任务**:
  1. 顶部导航栏（地图选址 + 搜索框）
  2. Banner轮播区
  3. 金刚区网格（8个3D图标入口）
  4. 同城头条资讯
  5. 密集服务橱窗（家政服务、维修安装、便民信息三大版块）
  6. 全局悬浮拨号球
  7. 滚动动效与交互

### Stage 4: 全部服务页开发
- **目标**: 实现左右双栏服务分类与工人列表
- **子任务**:
  1. 左侧分类侧边栏
  2. 右侧工人卡片列表
  3. 分类切换动画
  4. 工人详情页（预约、评价、服务项）

### Stage 5: 用户中心（"我的"）开发
- **目标**: 实现四种角色的用户中心
- **子任务**:
  1. 头部账户区（头像、昵称、角色标签）
  2. 身份切换弹窗（长按头像→选择→账号密码验证）
  3. 普通用户菜单（我的订单、浏览记录、客服、关于）
  4. 工人/师傅中心（数据概览、职业信息、接单大厅、收益统计）
  5. 合作商中心（发布信息、我的发布、收益统计、提现）
  6. 管理员后台（数据统计、用户管理、订单管理、内容管理、Banner管理）

### Stage 6: 订单与通讯系统开发
- **目标**: 实现完整订单流程和即时通讯
- **子任务**:
  1. 下单预约流程
  2. 抢单接单逻辑（云函数）
  3. 订单状态机（待接单→已接单→服务中→已完成→已评价）
  4. 即时通讯页面（消息列表、聊天界面、消息类型）
  5. 系统消息推送

### Stage 7: 发布与内容管理开发
- **目标**: 实现信息发布和审核流程
- **子任务**:
  1. 发布信息页（招聘求职、婚恋介绍两种表单）
  2. 我的发布管理（编辑、删除、刷新置顶）
  3. 审核流程（提交→审核中→通过/拒绝）
  4. Banner管理

### Stage 8: 素材准备与视觉优化
- **目标**: 生成/准备所有视觉素材
- **子任务**:
  1. 生成8个3D风格服务图标
  2. 生成用户默认头像
  3. 生成金刚区渐变背景
  4. 生成灰色人头占位图
  5. 全局样式统一和视觉微调

### Stage 9: 集成测试与部署
- **目标**: 完整测试和部署上线
- **子任务**:
  1. 全局功能联调
  2. 角色切换流程测试
  3. 订单全流程测试
  4. 云函数调试与优化
  5. 编写项目部署说明

## 技能加载计划
- Stage 1: 无需技能加载，自主分析
- Stage 2-9: 加载 vibecoding-general-swarm（通用编码编排）
- 不加载 vibecoding-webapp-swarm（该skill专用于React webapp，不适用小程序原生框架）

## 质量门控
- 每个阶段完成后必须输出可交付成果
- 每3个阶段进行一次全局检查点
- 最终交付物: 完整的小程序项目代码包 + 部署文档
